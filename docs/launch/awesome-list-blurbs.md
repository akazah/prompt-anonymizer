# awesome-list submission blurbs

One PR per list, after packages are published (lists prefer installable
projects). Follow each list's contributing rules (alphabetical order,
format, no marketing language). Suggested targets and ready-to-paste lines:

## awesome-privacy (e.g. `pluja/awesome-privacy`)

> - [Prompt Anonymizer](https://github.com/akazah/prompt-anonymizer) -
>   Replace PII with reversible labels before a prompt reaches an LLM;
>   detection runs fully on-device (browser/WebGPU, CLI, local proxy). MIT.

## awesome-llm-security (e.g. `corca-ai/awesome-llm-security`)

> - [Prompt Anonymizer](https://github.com/akazah/prompt-anonymizer):
>   on-device, reversible PII anonymization for LLM prompts — consistent
>   labels survive the round-trip and are restored locally; includes a
>   pre-commit/CI `scan` gate and an OpenAI-compatible anonymizing proxy.

## awesome-japanese-nlp (e.g. `taishi-i/awesome-japanese-nlp-resources`)

> - [prompt-anonymizer](https://github.com/akazah/prompt-anonymizer) -
>   日本語対応の PII 匿名化(人名 NER・郵便番号・チェックデジット検証付き
>   マイナンバー・日本の電話番号形式)。LLM 送信前にラベル置換、返答を
>   ローカルで復元。

## awesome-pii / data-anonymization lists

> - [Prompt Anonymizer](https://github.com/akazah/prompt-anonymizer) -
>   Reversible pseudonymization tuned for the LLM round-trip: same value →
>   same label, mapping never leaves the device. 10 languages. Python +
>   TypeScript cores with behavioural parity.

## awesome-chatgpt / awesome-llm-tools lists

> - [Prompt Anonymizer](https://github.com/akazah/prompt-anonymizer) -
>   Strip names, phone numbers and other PII from prompts before they reach
>   ChatGPT/Claude/Gemini and restore them in the reply — on-device browser
>   app, Chrome extension, CLI and OpenAI-compatible local proxy.

## Notes

- Check each list's last-merged PRs first; dormant lists (no merges in
  6+ months) are not worth the effort.
- Do not submit the same week to lists maintained by the same person.
- If a list requires a minimum star count, park it and revisit post-launch.
